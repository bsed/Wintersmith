Framer.js бесплатно на Windows и OS X

Антон Карташов • 04 марта 2016

Framer.js и Framer Studio — разные штуки. Framer.js — это бесплатная JavaScript-библиотека, которой можно пользоваться в любом редакторе кода и просматривать в обычном браузере. А Framer Studio — это платное приложение, заточенное под работу с этой библиотекой. Ниже я буду рассказывать, как с помощью редактора кода Atom пользоваться Framer.js бесплатно на Windows и OS X.

Эта статья — дополненный перевод статьи Кенни Чена Using Framer on Windows with Atom из его блога Prototyping with Framer

Что Framer Studio уже умеет, а нам придется настроить:

  1. Синхронизацию изменений с браузером.
  2. Конвертацию CoffeeScript в JavaScript.
  3. Публикацию в интернет, чтобы посмотреть прототип на устройстве либо отправить коллегам.
  4. Превью прототипа с мокапами телефонов, руками людей и красивыми фонами.

Что умеет Framer Studio, а мы без приложения не сможем сделать:

  • Бесплатный хостинг на share.framerjs.com. Нам придется использовать свой ftp-сервер либо GitHub Pages для публикации прототипа, если мы хотим показать его кому-то еще.
  • Подсветку кода и подсказки со встроенной документацией. Любой редактор кода умеет хорошо подсвечивать CoffeeSctipt, но ни один из них не знает компонентов и функций библиотеки Framer.js – их придется выучить. Благо документация доступна на официальном сайте Фреймера.
  • Синхронизацию с мобильными приложениями (iOS, Android). Если ваш Мак и Айфон подключены к одному вай-фаю, то приложение видит все открытые во Framer Studio проекты и подгружает актуальные изменения.

1. Установка Атома

Атом — редактор кода от GitHub, у которого есть визуальный интерфейс для установки плагинов (в <Sublime Text, например, все настройки хранятся в config-файлах и меняются кодом. Это очень удобно для разработчиков и больше никого).

2. Установка coffee-compile и html-preview

Опытным разработчикам очень не нравится, что Framer Studio не позволяет писать на чистом JavaScript и Babel, но если вы новичок в программировании я очень рекомендую начать с CoffeeScript: вам придется меньше сталкиваться со знаками препинания, и CoffeeScript намного лаконичнее само по себе:

/* JavaScript */
for (var i = 0; i < 10; i++) {
  function hello() {print("Hello world!");
}

# CoffeeScript
for [0..10]
  hello = -> print "Hello world!"

3. Создание нового проекта

Каждый новый проект создается на основе шаблона, который можно скачать в репозитории Framer на GitHub. Он представляет собой папку с файлами:

  • index.html отображает работу прототипа в браузере (поддерживает только WebKit: Chrome, Safari, Яндекс.Браузер).
  • app.js / app.coffee содержат код, который мы пишем.
  • /framer — папка, которая содержит старую версию библиотеки (позже мы ее заменим на новую).
  • /images — сюда нужно складывать нарезку вашего макета.
